PRODUCTION-READY USER INTERFACES
=================================

Your English-Luganda Translator now has TWO professional user interfaces, both fully tested and production-ready.


OPTION 1: STREAMLIT WEB APP (Recommended for End Users)
=======================================================

File: app_streamlit_ui.py

Start Command:
    streamlit run app_streamlit_ui.py

Access:
    http://localhost:8501 (automatically opens in browser)

Features:
    ✓ Modern, beautiful web interface
    ✓ Bidirectional English ↔ Luganda translation
    ✓ Auto-language detection
    ✓ Real-time translation with confidence scoring
    ✓ Translation history with database persistence
    ✓ Common phrases & phrasebook with categories
    ✓ Recent translations table with export
    ✓ System information & settings
    ✓ Clear interface for mobile & desktop
    ✓ Responsive design

Interface Layout:
    Left Panel:
        - Input text area
        - Language selection (auto-detect, English, Luganda)
        - Translate button

    Right Panel:
        - Translation output
        - Confidence percentage
        - Direction indicator

    Tabs:
        1. Translation History - View recent 20 translations with export to CSV
        2. Phrasebook - Learn common phrases by category
        3. About - System information and features
        4. Settings - Configuration and system status

Performance:
    First translation: ~30-60 seconds (model loading)
    Subsequent: ~1-3 seconds (model cached)
    GPU: ~1 second per translation
    CPU: ~3-5 seconds per translation


OPTION 2: FLASK REST API (Recommended for Developers)
======================================================

File: app_flask_api.py

Start Command:
    python app_flask_api.py

Access:
    Web Interface: http://localhost:5000
    API: http://localhost:5000/api/*

Features:
    ✓ RESTful API with JSON responses
    ✓ Single text translation
    ✓ Batch translation (multiple texts)
    ✓ Language detection
    ✓ Translation history storage & retrieval
    ✓ Phrasebook/learning content delivery
    ✓ System status endpoint
    ✓ HTML web interface for browser access
    ✓ Comprehensive error handling
    ✓ Thread-safe operations

API Endpoints:

1. POST /api/translate
   Translate single text
   Request:
       {
           "text": "Hello, how are you?",
           "source_lang": "english"  (optional)
       }
   Response:
       {
           "source_text": "Hello, how are you?",
           "target_text": "Oli otya?",
           "source_lang": "english",
           "target_lang": "luganda",
           "confidence": 0.87,
           "timestamp": "2026-05-14T10:30:00"
       }

2. POST /api/batch-translate
   Translate multiple texts at once
   Request:
       {
           "texts": [
               "Hello",
               "How are you?",
               "Thank you"
           ]
       }
   Response:
       {
           "results": [
               {...},
               {...},
               {...}
           ],
           "count": 3
       }

3. POST /api/detect-language
   Detect if text is English or Luganda
   Request:
       {"text": "Oli otya?"}
   Response:
       {
           "text": "Oli otya?",
           "detected_language": "luganda",
           "timestamp": "2026-05-14T10:30:00"
       }

4. GET /api/history?limit=20
   Get translation history
   Response:
       {
           "results": [
               {
                   "timestamp": "2026-05-14T10:30:00",
                   "source_lang": "english",
                   "source_text": "Hello",
                   "target_text": "Habari",
                   "confidence": 0.90
               },
               ...
           ],
           "count": 20
       }

5. GET /api/phrasebook
   Get common phrases organized by category
   Response:
       {
           "greetings": [
               {"english": "How are you?", "luganda": "Oli otya?"},
               ...
           ],
           "family": [...],
           "daily": [...]
       }

6. GET /api/status
   Get system and model status
   Response:
       {
           "status": "running",
           "device": "cuda|cpu",
           "model": "models/trained_nllb_enhanced/final",
           "timestamp": "2026-05-14T10:30:00"
       }

7. DELETE /api/clear-history
   Clear all translation history
   Response:
       {"status": "cleared"}

Example API Usage:

    # Python requests
    import requests
    r = requests.post('http://localhost:5000/api/translate',
                      json={'text': 'Hello world'})
    print(r.json())

    # cURL
    curl -X POST http://localhost:5000/api/translate \
         -H "Content-Type: application/json" \
         -d '{"text": "Hello world"}'

    # JavaScript fetch
    fetch('http://localhost:5000/api/translate', {
        method: 'POST',
        headers: {'Content-Type': 'application/json'},
        body: JSON.stringify({text: 'Hello world'})
    }).then(r => r.json()).then(console.log)

TROUBLESHOOTING
===============

Issue: Port already in use
    Streamlit: streamlit run app_streamlit_ui.py --server.port 8502
    Flask: Edit app_flask_api.py, change port in last line

Issue: Module not found
    pip install streamlit flask torch transformers

Issue: Model loading slowly (first run)
    Normal! Model downloads ~2.5GB automatically
    Subsequent runs use cached model (fast)

Issue: CUDA/GPU errors
    Automatically falls back to CPU
    Works fine, just slower

Issue: 404 Not Found on Flask
    Make sure templates/index_production.html exists
    Check Flask is actually running on port 5000

Issue: Translation looks bad
    Train the model: python train_nllb_enhanced.py --epochs 5
    Current uses base model or existing trained model

PERFORMANCE OPTIMIZATION
========================

Fast Translation (1 second):
    ✓ Use GPU
    ✓ Use batch processing
    ✓ Model already cached
    ✓ Short texts (< 256 tokens)

Slow Translation (10+ seconds):
    ✓ Using CPU
    ✓ First run (model download/load)
    ✓ Very long texts
    ✓ No GPU available

To Speed Up:
    1. Add GPU: nvidia-docker, CUDA 11.8+
    2. Use batch: API batch endpoint
    3. Optimize prompts: Keep texts short
    4. Cache model: Uses automatic caching
